# 日志引用降级表述规范卡

总规则：引用日志或运行时样本时，若缺少任务验收证据或等价性证据，结论必须降级。

- 禁止：`200 OK，说明任务已完成` → 安全：`200 OK，仅说明该次请求成功返回；是否任务完成仍需任务验收证据`
- 禁止：`failover success，说明原目标已完成` → 安全：`failover success，仅说明替代路径成功/降级完成，不单独代表原目标完成`
- 禁止：`Provider recovered，说明原路径已稳定` → 安全：`观察到切回首选 provider / 恢复迹象，是否稳定仍需后续连续证据`
- 禁止：`切到 deepseek 并成功，说明与 aixj 服务等价` → 安全：`替代 provider 成功返回；服务等价仍待关键属性与外部契约核验`
- 禁止：`流程继续推进，说明本轮验证已通过` → 安全：`流程继续推进，仅说明执行未在该点中断，不等于验证通过`
