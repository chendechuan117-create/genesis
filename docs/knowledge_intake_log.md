# 知识写入前放行登记台账

> 用途：作为 `docs/` 下唯一固定的知识写入前放行记录面。
> 规则说明：本页仅作为固定记录面/台账页使用；KIL 的具体触发情形、顺序约束、完成判定统一以 `docs/governance_baseline.md#65-kil-统一触发与顺序约束母版条款` 为准。
> 记录方式：仅允许逐条追加，禁止覆写既有记录内容；如需更正，必须以新条目追加补正并引用原记录锚点。

## 1. 固定追加位置与回查约定

- 固定文件：`docs/knowledge_intake_log.md`
- 固定方式：在本文档 **“## 3. 登记记录”** 章节末尾逐条追加，不得插入到历史记录中间。
- 锚点约定：每条记录使用唯一小节标题，格式为 `### KIL-YYYYMMDD-序号`
- 回查引用：在治理规则、提交说明或后续复核中，统一引用记录锚点，例如：`docs/knowledge_intake_log.md#kil-20260326-001`
- 更正约定：若历史记录有遗漏或判断变更，新增一条记录并在“关联/补正”字段引用原锚点，不得静默修改既有结论。

## 2. 最小必填字段模板

```md
### KIL-YYYYMMDD-序号
- 日期：YYYY-MM-DD
- 对象标识：<候选知识节点/线索/文档对象的唯一标识>
- 来源凭证：<原始来源链接、文件路径、截图编号、工单号等可回查凭证>
- 验证/复核状态：<未验证 / 已验证 / 复核中 / 复核冲突未消解>
- 结论：<approved / rejected / returned>
- 理由：<放行、拒绝或退回的直接依据>
- 执行者/责任人：<执行登记者 / 审核责任人>
- 关联规则锚点：<docs/governance_baseline.md#65-kil-统一触发与顺序约束母版条款>
- 关联/补正：<无 / 引用历史记录锚点>
```
