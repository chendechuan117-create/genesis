# 点线面（Point-Line-Surface）

> Genesis V4 的知识架构。替代圆锥模型。
> 工程实施细节见 `point_line_surface_implementation.md`。

---

## 动机

LLM 没有内部记忆。每次调用，上下文窗口打开，它诞生；窗口关闭，它死亡。任何不在窗口里的东西，对它而言都不存在。

但 LLM 需要理解一个复杂的代码库——信息量远超上下文窗口容量。它需要在每一轮"新生"时，瞬间继承所有历史探索的成果，知道哪里是稳固的基础，哪里是未探索的前沿，哪里是反复修补的伤疤。

点线面架构就是这个继承系统。它不是知识库，不是代码压缩，不是依赖图。它是 LLM 在探索过程中用推理链编织的一张外部菌丝网络。这张网络让一个只有"此刻"的智能体，在出生的瞬间就能一窥全貌。

圆锥模型把所有维度压成一个 confidence 数字来衡量价值，跟实际脱节，无法调试。点线面用拓扑替代数字——价值不是算出来的，是从结构中涌现的。更准确地说：不是用入线数替代 confidence，而是让每个维度独立表达，拓扑作为主信号。

---

## 点（Point）

点 = LLM 产生并读取的认知片段。不是"这段代码做什么"的客观描述，而是"我读到这段代码时，我理解了什么"。

由此，点天然具备以下性质：
- **片面性**：它只捕捉当时 LLM 所见的那个角度
- **可错性**：它记录的是一种理解，而理解可能是错的
- **语境依附**：它诞生于某一次具体的探索，带着当时的上下文印记

不追求客观完整，追求的是"那一刻的真实"。

### 来源属性（provenance）

同样是"旧点"或"0入线点"，可能来自不同时期，不能简单等同为未探索前沿：

- **GP 新点**：本轮推理产生
- **legacy 旧点**：点线面部署前由旧系统留下，可能有价值但未被 reasoning_lines 连入拓扑
- **human seed**：人工注入的冷启动坐标，先是导航参考，之后可能被 GP 引用成推理 basis
- **reconnected**：旧点被新线重新引用，重新进入推理网络

> **0 入线 ≠ 无价值。** legacy/seed 节点可能是整个知识图谱的根锚点。
> 入线数低只意味着"当前推理前沿不经过它"，不意味着"它错了"。
> 正确动作不是降权，而是等待或触发重连：让新的 GP 在验证它时写出新点，用线把它接回网络。

## 线（Line）

线 = 推理依赖关系，不是数字权重。

线记录的不是代码的依赖关系（A 调用了 B），而是推理的依赖关系："我之所以产生新认知 B，是因为我基于旧认知 A"。

- 新点 → 旧点的线："我基于已有的某个认知，产生了这个新洞察。"（**异轮**线贡献旧点入线数，同轮线不贡献）
- 新点 → 新点的线："我这个新洞察，是基于同轮中另一个新发现产生的。"（记录因果但标记为同轮线）

### 同轮隔离

LLM 在单次调用中连续推理，产生的链条如果每条都算数，就等于"我引用我自己当证人"。这是循环论证。

因此，同轮线不贡献入线数。只有跨调用——完全不同的 LLM 实例，在完全不同的上下文里，独立踩上了同一条推理路径——才算一次真正的验证。

**同轮隔离不是防御性的工程约束，而是维持信任测量有效性的唯一前提。**

### 因果 vs 入线数

线拆分为两个概念：

- **因果（写入/审计可见）**：推理链"基于什么产生"，每条线的 reasoning 回答一个不同的因果问题
- **入线数（GP 不可见）**：点被多少异轮新点基于它产生，仅供后台判断价值

GP 写入新线时必须声明因果理由，系统审计时可以追溯 reasoning；但常规搜索和面默认不把完整 reasoning 文本注入给 GP，只暴露角色标签（基础/探索）和必要的定性标记。GP 不看到精确入线数——否则会把入线数当 confidence 用，回到圆锥模型老路。

### 碰撞检测

GP 连线到 A,B,C → 发现已有节点也引用 A,B,C → 碰撞提醒。

- 碰撞只提醒，不自动判定（引用相同素材可以写出完全不同的结论）
- 重复碰撞累积为饱和标记（见拓扑信号→密度信号）
- 当前 MVP：basis 集合重叠 = 碰撞信号，不需要推理链语义比较

> 碰撞检测如何消费增长信号、如何作为拓扑密度的观察窗口，见"拓扑消费机制→碰撞检测"。

### 采集方式

GP 优先通过低摩擦工具采集：先用 `record_point` 写新点，再用 `record_line` 为每个依据节点写因果线。没有线的创新 = 无法判断价值 = 无法去重 = 噪音。

## 面（Surface）

面 = 搜索后的当轮上下文组合。不是持久结构，每轮重新装配。

- 面用线的拓扑结构（入线数决定 BFS 流向），但常规面不展示完整 reasoning 文本
- 面是搜索的后处理层，不替代搜索

```
search_knowledge_nodes → 找到种子点 A, B
  ↓
expand_surface(seed_ids, context_budget) → BFS 扩散组装面
  ↓
GP 看到：知识网络 + 前沿在哪 + 空洞在哪 + 点的角色（基础/探索）
```

### 两阶段组装

**阶段一：填充** — 从种子点出发，优先队列按入线数排序遍历邻居。reasoning_lines 只沿 new→old 方向（走向已验证基础），node_edges 双向走。入线数高的路径优先扩展 = 已被反复验证的推理通道 = 可靠的推理基础。

**阶段二：推进** — 逐步将填充中的高入线数节点替换成前沿的低入线数节点。效果：GP 被推向知识前沿。

先填充基础，再推向前沿。不是同时生效。

### 点的角色标签

- **基础**：入线数 ≥ P75，已被反复验证，可靠的推理基础
- **探索**：入线数 < P75，新近产生，尚未被验证的前沿

阈值用分布分位数（P75）而非固定值，随知识库规模自适应：库小时阈值低，库大时阈值自动升高。

> 面如何消费三个拓扑信号来决定组装行为，见"拓扑消费机制→面组装"。

## 拓扑信号

PLS 回答的不是单个节点评分问题，而是：**哪些点、路径、区域值得进入 GP 当轮有限上下文？**

点是否可见、路径如何扩展、区域是否饱和、当轮面如何装配，都是同一个裁决问题的不同投影。

旧系统用 confidence 数字回答。PLS 用拓扑回答——价值不是算出来的，是从结构中涌现的。拓扑信号有三个维度，它们回答的是同一个问题的三个面：

### 增长信号：入线数

入线数 = 点被多少个"异轮"新点基于它产生 = 这个点作为推理依据，被后来的探索者独立踩过多少次。

入线数不是"这个点说得好"的评分，不是受欢迎程度，不是点赞数。它是"这个点被后来的探索者反复踩过"的频次记录。被踩的次数多了，自然就成了路。

由此，入线数标记的往往不是正确答案，而是**反复被拆、反复被补的那面墙**。它高，不是因为设计得好，而是因为它处于冲突和修补的中心。

- 回答的问题：这个节点**正在**被依赖吗？
- GP 看到：角色标签（基础/探索）

### 衰减信号：CONTRADICTS 边

当旧节点被新知识显式否定时，CONTRADICTS 边被连上。这不是谁的主观判断，而是拓扑中新增的一条边——和入线数一样，是从结构中涌现的。

高入线 + 有 CONTRADICTS = "曾经重要，正在被取代"。这是 PLS 替代 confidence 时间衰减的拓扑衰减：节点不是因为"老了"而贬值，而是因为"被否定了"而贬值。

- 回答的问题：这个节点**正在**被取代吗？
- GP 看到：⚔️矛盾标签
- 消融候选信号：有 CONTRADICTS + 高入线 = 优先消融

### 密度信号：饱和标记

碰撞检测发现同一区域反复重叠 → 产生饱和标记。不是认知片段，而是"该区域反复发生路径重叠"的拓扑密度信号。

饱和标记不是"这里绝对没有新东西"的判决，而是一个密度假设：继续在该区域探索的边际收益可能下降，应把注意力推向不饱和区域。

- 回答的问题：这个区域**是否正在接近饱和**？
- GP 看到：饱和倾向提示（"该区域路径重叠频繁/可能已接近饱和"）
- 面组装时：疑似饱和区域的节点降权（0.5×），引导 GP 去不饱和区域
- 碰撞后由系统自动创建/递增，GP 不需要也不应该手动创建；单次标记只是密度样本，重复累积才形成饱和判断

### GP 可见性边界（不变量）

GP 只能看到定性标签（基础/探索/⚔️矛盾/饱和/有实战记录），不能看到定量数字（入线数/fusion_score/win_rate）。

这个约束和同轮隔离是同一性质——都是防止 PLS 退化的结构性约束：

- **同轮隔离**防止：GP 自刷入线数 → 入线数退化成自证游戏
- **可见性边界**防止：GP 追逐数字 → 回到 confidence 数字崇拜

如果 GP 看到入线数=5，它会开始想"我要引用入线数高的节点"而不是"我要引用推理上合理的节点"。数字会替代判断。

## 拓扑消费机制

拓扑信号是 PLS 的核心，但信号本身不做任何事。以下机制**消费**拓扑信号来决定系统行为：

### 面组装（消费增长+衰减+密度信号）

面从拓扑中切出当轮上下文切片（面的本体定义见"面（Surface）"章节）。三个信号同时生效：

- 增长信号（入线数）→ BFS 方向：走高入线路径 = 踩稳基础
- 衰减信号（CONTRADICTS）→ 排除矛盾节点 = 不把 GP 踩到正在被取代的旧知识上
- 密度信号（饱和标记）→ 疑似饱和区域降权 = 降低 GP 在高重叠区域空转的概率
- RELATED_TO 边 → BFS 跨锥体连接：让面能从不同知识锥体间跳转，增加上下文多样性

面的生命周期：search → expand → inject → discard。每轮重建，不跨轮累积。拓扑是永久的结构，面是每轮从拓扑中投影出的临时切片。

### 碰撞检测（消费增长信号）

GP 连线到 A,B,C → 发现已有节点也引用 A,B,C → 碰撞提醒。只提醒不阻止——引用相同素材可以写出完全不同的结论。

碰撞是增长信号的一个观察窗口：basis 集合重叠度高 = 多个推理者走了同一条路 = 该区域拓扑密度高。

### 必要性消融（消费增长+衰减信号）

消融不是独立机制，是拓扑信号的**自校正**：拓扑提出假设（高入线 + 有 CONTRADICTS = "这个节点可能过时了"），消融做实验验证（隐藏节点，观察系统行为），结果修正拓扑可见性。

- **向前**（LLM 内部已有此知识）→ 降级旧节点
- **向后**（系统仍依赖旧节点）→ 恢复可见
- 消融节点从所有 GP 可见通道统一隐藏（7 通道过滤）

## 互补信号与信息系统分层

拓扑信号回答"点、路径、区域在当前网络中处于什么结构位置"。但还有其他维度回答不同的问题——它们不是 PLS 的竞争者，因为它们回答的根本不是同一个问题：

| 信号 | 回答的问题 | 与 PLS 的关系 |
|---|---|---|
| 入线数 + CONTRADICTS + 饱和 | 点/路径/区域的**拓扑状态**是什么？ | PLS 核心 |
| Arena 胜率 | 这个节点被使用时**结果如何**？ | 互补：拓扑说位置，Arena 说后果 |
| trust_tier | 这个节点**从哪来**？ | 正交：来源可靠性，与拓扑价值无关 |
| signature | 这个节点**适用什么环境**？ | 正交：适用范围，与价值无关 |
| fusion_score | 搜索结果**怎么排序**？ | 内部：排序机制，GP 不可见 |

旧系统的问题不是"用了 confidence"，而是**把所有维度压成一个数字**。PLS 的核心不是"用入线数替代 confidence"，而是让每个维度独立表达，拓扑作为主信号，其他作为互补/正交/内部信号。

### 信息系统分层

PLS 是价值架构，不是整个元信息系统。其他子系统有各自的目标，与 PLS 协作但不属于 PLS：

- **PLS 本体**：点、线、面、拓扑信号、同轮隔离、可见性边界
- **PLS 消费机制**：面组装、碰撞检测、必要性消融
- **PLS 周边系统**：vector（语义召回）、signature（环境匹配）、trace（程序性记忆）、arena（后果记录）——提供召回/环境/后果/经验，但不拥有价值裁决权

## 自洽性检验

从"LLM 没有状态"这一条约束出发，整个架构是逐步推论后的必然产物：

- 没有内部记忆 → 记忆必须在外部
- 外部记忆不能只存原始经验 → 必须压缩
- 压缩不能丢逻辑关系 → 依赖必须记录为推理链
- 推理链的信任不能来自同一实例 → 同轮不计数
- 信任累积导致惯性 → 填充之后必须推进，必要时必须遗忘
- 遗忘不能盲目 → 需要衰减信号（CONTRADICTS）指示哪些节点正在被取代
- 探索不能重复 → 需要密度信号（饱和标记）指示哪些区域可能接近饱和
- 一切操作最终不是为了存储更多信息，而是让无状态的 LLM 在出生的瞬间，找到抵达问题本质的最短路径

每一对机制彼此咬合：点的定义与线的定义匹配（足迹需要推理链来追溯价值），入线数与必要性消融匹配（一个测频次，一个测必要性），增长信号与衰减信号匹配（入线数说"被依赖"，CONTRADICTS 说"被取代"，消融验证两者的张力），面的填充与推进匹配（先踩稳再探索），碰撞检测与同轮隔离匹配（用跨实例验证来去重），可见性边界与同轮隔离匹配（都是防止 PLS 退化的结构性约束）。没有互相矛盾的定义。

## 与圆锥模型对比

| 维度 | 圆锥模型 | 点线面 |
|---|---|---|
| 价值信号 | confidence 数字 | 拓扑信号（增长+衰减+密度） |
| GP 看到的 | 数字评分 | 定性标签（基础/探索/⚔️矛盾/饱和） |
| 可调试性 | 黑盒 | 可观测：拓扑结构可见 |
| 去重 | 内容语义相似度 | basis 集合重叠 |
| 面 | 凝实度排序 | 两阶段 BFS（踩稳+推进） |
| 消融 | win_rate 低 → 淘汰 | 拓扑信号自校正（增长+衰减 → 验证必要性） |
| 衰减 | 时间衰减 | 拓扑衰减（CONTRADICTS 边） |
| 信号关系 | 所有维度压成一个数字 | 每个维度独立表达，拓扑为主信号 |
