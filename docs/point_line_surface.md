# 点线面元信息架构设计

> 2026-04-22 讨论确认，替代圆锥模型

## 核心动机

圆锥模型最大问题：**无法感受信息是否有价值**。数字评分（confidence/凝实度）跟实际价值脱节，导致无法调试、无法修改。点线面用拓扑替代数字，让价值可观测。

---

## 点（Point）

- = LESSON / 锚点，LLM写，LLM读
- 分新点和旧点，判断只留存于一轮
- 新点 = 本轮探索中产生的新知识
- 旧点 = vault中已有的知识节点

---

## 线（Line）

### 定义
线 = **判断依据 / 推理链片段**，不是数字权重。

- 新点 → 旧点的线："我为什么觉得旧LESSON有用"
- 新点 → 新点产生的线："新洞察基于什么想法/观察产生"

### 存储
- **独立表/平台**，不与点和面混存
- 点 → 面不经过线（面不需要因果关系）

### 线拆分为两个概念
- **因果（GP可见）**：推理链"基于什么产生"，GP可以知道因果关系
- **被调用数量（GP不可见）**：入线数，仅供后台调试/判断价值，防止马太效应

### 用途
1. **推理链记录**：记录新点基于什么产生
2. **价值信号**：入线数 = 点被多少新点基于它产生（后台用，GP不可见）
3. **去重/换皮筛查**：两个新点如果"基于什么产生"的线相似，可能是换皮（线相似度，非内容语义相似度）

### 采集方式
**GP连线**：GP通过 `record_lesson_node` 的 `reasoning_basis` 参数主动连线。
- GP搜索到旧LESSON → 基于它产生新洞察 → 记录时声明"基于哪个节点、为什么觉得有用"
- C-Phase同理：回顾GP执行时，也用 `reasoning_basis` 记录推理链
- 线 = GP推理过程的因果记录，不是C-Phase事后补充

---

## 面（Surface）

### 定义
面由点组合而成，按推导链组合成网。

- A推导出B，B推导出C → A+B+C组成面
- 非线性，像网一样扩散
- 面不需要知道因果关系，只需要知道"有什么"

### 液体扩散机制
1. 从种子点出发，沿推导链BFS向外扩散
2. 扩散填充上下文，像液体蔓延
3. 到上下文20%时进入替换阶段

### 摸石头过河（替换策略）
- 逐步将起点的点替换成边缘的点（新点）
- 暂定策略：踢掉核心/最旧的点，加入边缘邻居
- 效果：**GP始终看到知识前沿，而非核心旧知识**

### 冷启动
- 种子点 = 搜索命中节点（top 8），不需要GP额外决定
- 不需要特殊冷启动机制

---

## 面与搜索的关系

面是搜索的**后处理层**，不替代搜索。

```
search_knowledge_nodes → 找到种子点 A, B
  ↓
expand_surface(seed_ids, context_budget) → 沿推导链BFS扩散
  ↓
GP看到：知识网络 + 前沿在哪 + 空洞在哪（没有数字）
```

替代现有凝实度摘要和confidence显示。GP从网络拓扑自己感受价值，不读数字。

---

## 与圆锥模型的对比

| | 圆锥模型 | 点线面 |
|---|---|---|
| 价值信号 | confidence数字评分 | 入线数（拓扑） |
| 给GP看什么 | 凝实度摘要（数字） | 知识网络（拓扑） |
| 可调试性 | 黑盒，调参数看效果 | 可观测：没有入线=没人基于它推理过 |
| 去重方式 | 内容语义相似度（0.85阈值） | 线相似度（推理链相似度） |
| GP定位 | 看核心/高置信知识 | 始终在知识前沿 |
